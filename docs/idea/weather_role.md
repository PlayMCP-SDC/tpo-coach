# 계절 필터 설계 — 소매기장 + 소재 보온등급 (weather_role)

> 상태: **설계 확정(브레인스토밍 결과)**. 코디 추천에 계절 정보가 들어왔을 때 계절에
> 맞지 않는 코디를 **하드 필터**로 제외하는 규칙의 단일 출처. 구현 계획은 다음 단계(writing-plans).

## 1. 결정 요약

- **계절 3버킷:** `봄가을` · `여름` · `겨울`. 요청당 계절은 하나.
- **동작:** 하드 필터. 코디(세트)의 **어느 한 부위라도** 그 계절 금지 조건에 걸리면 **코디 전체 탈락**.
- **필터 신호(중요도순):**
  1. **소재 → 보온등급**(따뜻/시원/중립) — 여름·겨울 판단의 주력.
  2. **소매기장**(반팔/민소매/캡/긴팔/7부/없음) — 겨울에서 팔 노출템 제외용.
  3. **기장(옷 길이)** — 맥시(김)·미니(짧음) 극단만 사용.
  4. **카테고리** — 아우터 `코트·패딩` 등 명백한 계절템.
- **로직 위치:** 쿼리 시점 계산(WHERE 절). 계절→금지조건은 파이썬 규칙 dict 한 곳에서 관리.
  (실험 단계라 재빌드 없이 규칙 튜닝 가능. 안정화되면 빌드 시점 태깅으로 굳혀도 됨.)
- **안 쓰는 필드:** 디테일·넥라인·핏·프린트·색상·옷깃 — 계절 신호 약함(YAGNI).

## 2. 스키마 변경 (재빌드 1회 필요)

기존 `outfits` 테이블에 컬럼 추가. 원천 라벨링 JSON의 부위별 `소매기장`·`소재`에서 도출.
**원값 보존 + 필터용 파생값 사전 계산** 방침(옵션 2).

| 컬럼 | 대상 부위 | 값 | 비고 |
|---|---|---|---|
| `{part}_sleeve` | 상의·아우터·원피스 | 소매기장 **원값** | **하의엔 소매기장 없음** → 컬럼 없음 |
| `{part}_material` | 상의·하의·아우터·원피스 | 소재 **원값**(리스트) | 손실 없이 보존. 미래 기능용 |
| `{part}_warmth` | 상의·하의·아우터·원피스 | `따뜻`·`시원`·`중립` | `material`에서 **파생 계산**(축약) |

- `top_sleeve`, `outer_sleeve`, `dress_sleeve`
- `top_material`, `bottom_material`, `outer_material`, `dress_material`
- `top_warmth`, `bottom_warmth`, `outer_warmth`, `dress_warmth`

**소재 원값 저장 형태:** 원천 `소재`는 리스트(다중값). SQLite엔 리스트 타입이 없으므로
`"우븐,린넨"` 형태의 구분자 문자열 또는 JSON 문자열로 보관(구현 시 확정). `warmth`는
이 원값에서 3장 우선순위로 빌드 시점에 미리 구워 별도 컬럼에 저장.

**역할 분리:** 계절 필터는 파생값 `{part}_warmth`·`{part}_sleeve`·기장·카테고리만 조회.
`{part}_material` 원값은 이번 필터에선 안 쓰고 미래 기능(소재 기반 추천 등)을 위해 보존만.

## 3. 소재 → 보온등급 매핑표 (25종)

한 부위의 `소재`는 리스트(다중값). **축약 우선순위: 따뜻 > 시원 > 중립**
(하나라도 따뜻이면 따뜻, 없으면 시원 하나라도 있으면 시원, 아니면 중립). 소재 없음(NULL) → `중립` 취급.

| 등급 | 소재 |
|---|---|
| **따뜻** (겨울/가을) | 울/캐시미어, 트위드, 퍼, 무스탕, 플리스, 패딩, 코듀로이, 스웨이드, 헤어 니트, 벨벳, 네오프렌, 가죽 |
| **시원** (여름) | 린넨, 시폰, 메시, 레이스, 실크 |
| **중립** (사계절) | 우븐, 니트, 저지, 데님, 스판덱스, 자카드, 시퀸/글리터, 비닐/PVC |

> 설계 근거:
> - `니트`(소재)는 두께가 다양해 **중립**. 겨울 니트 신호는 카테고리 `니트웨어`+소매기장(긴팔)이 담당.
> - `헤어 니트`는 명백히 두꺼워 **따뜻**.
> - `가죽`은 여름 부적합이라 **따뜻** 쪽(여름 제외 목적). 봄가을엔 통과.
> - `자카드`는 직조 두께가 다양해 보수적으로 **중립**.

## 4. 계절별 최종 규칙표 (금지 = 코디 탈락)

각 계절에서 아래 조건 중 **하나라도** 해당하는 부위가 있으면 코디 전체를 제외한다.

### ☀️ 여름
| 신호 | 조건 |
|---|---|
| 보온등급 | 임의 부위 `warmth = 따뜻` |
| 카테고리 | `outer_category ∈ {코트, 패딩}` |
| 기장 | `bottom_length = 맥시` 또는 `dress_length = 맥시` |

- 소매기장은 **제외 안 함** — 긴팔이 데이터 대다수(22k)이고 얇은 긴팔 여름옷 존재.
- 점퍼·니렝스·니트웨어 애매함은 보온등급(따뜻 제외)이 처리.

### ❄️ 겨울
| 신호 | 조건 |
|---|---|
| 보온등급 | 임의 부위 `warmth = 시원` |
| 소매기장 | `sleeve ∈ {민소매, 반팔, 캡}` (상의·원피스·아우터) |
| 기장 | `bottom_length = 미니` 또는 `dress_length = 미니` |
| 카테고리 | `top_category = 브라탑` |

- `7부소매`·`없음`·`긴팔`은 통과. `없음`은 의미 모호 → 보수적으로 통과.
- 크롭 상의·탑(민소매)은 소매기장/보온등급으로 자연 처리.

### 🍃 봄가을 (가장 관대 — 혹한/혹서 전용템만 제외)
| 신호 | 조건 |
|---|---|
| 카테고리 | `outer_category = 패딩` |

- 보온등급으로는 거의 안 거름. `울코트`(따뜻)·`린넨`(시원) 모두 봄가을 통과.
- 근거: `린넨 상의 + 울코트` 같은 혼합 코디는 여름(따뜻 제외)·겨울(시원 제외) 양쪽에서 빠지고
  **봄가을에만 남는다** → 세트 단위 보온등급 필터가 간절기 룩을 자연스럽게 라우팅.

## 5. 엣지 케이스

- **NULL 처리:** `sleeve`·`warmth`·`length`가 NULL/빈값이면 그 조건은 **통과**(데이터 결측으로 코디를 떨어뜨리지 않음, 보수적).
- **다중 소재:** 3장 우선순위(따뜻>시원>중립)로 등급 1개 확정.
- **세트 판정:** 부위 조건은 OR로 결합 — 한 부위라도 걸리면 탈락(하드 필터 취지).
- **후보 고갈:** 필터 후 표본이 부족하면 현행처럼 "찾지 못했습니다" 안내(하이브리드 완화는 이번 범위 아님).

## 6. 이번 범위 아님 (Out of scope)

- 소재 원값(`material`)을 **필터/노출에 사용** — 이번엔 보존만, 계절 필터는 `warmth`만 사용.
- 디테일·넥라인·핏·프린트 등 순수 스타일 속성 저장/필터.
- 간절기 하이브리드 완화(부족 시 근접 계절로 채우기).
- 빌드 시점 season 태깅(규칙 안정화 후 검토).
