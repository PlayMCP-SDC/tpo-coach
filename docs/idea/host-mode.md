# 모임장 모드 — 단톡방 복장 가이드

모임 주최자가 상황을 자연어로 입력하면, 단톡방에 공유할 수 있는 **TPO 가이드 카드**를
만들어 주는 기능. 참석자는 카드를 보고 각자 착장을 맞추고, 필요하면 자기 옷만 비공개로 검사한다.

## 흐름 한눈에

```
모임장: 상황 입력(자연어)
   │
   ▼
MCP: 권장 드레스코드 판단 → 공유용 가이드 카드 생성
   │
   ▼
단톡방 공유
   │
   ▼
참석자: 카드의 check_criteria(기준) → 본인 착장을 텍스트로 묘사(또는 공개 image_url)해 비공개 검사
```

## 1. 모임장 입력 (예시)

> "이번 주 토요일 저녁 6시 호텔 연회장에서 졸업 모임이 있어.
> 너무 격식 차리지는 않되 운동복은 피했으면 좋겠어."

입력에서 추출하는 정보:

- **일시** — 요일/시간 (예: 토요일 저녁 6시)
- **장소** — 격식 수준에 영향 (예: 호텔 연회장)
- **성격** — 모임 종류 (예: 졸업 모임)
- **제약** — 주최자가 명시한 조건 (예: 격식 ↓, 운동복 ✕)

## 2. 생성되는 가이드 카드 (예시)

```
우리 모임 TPO 가이드

일시: 6월 27일 오후 6시
장소: ○○호텔 연회장
권장 수준: 스마트 캐주얼

추천
- 셔츠, 블라우스, 단정한 티셔츠
- 슬랙스, 스커트, 깔끔한 데님
- 단정한 운동화 또는 구두

피하면 좋은 것
- 운동복
- 슬리퍼
- 지나치게 편한 홈웨어

[내 옷 확인하기]
```

카드 구성 요소:

| 항목 | 내용 |
| --- | --- |
| 일시 / 장소 | 입력에서 추출한 모임 정보 |
| 권장 수준 | 드레스코드 한 단어 (예: 스마트 캐주얼) |
| 추천 | 상의 / 하의 / 신발 등 권장 아이템 |
| 피하면 좋은 것 | 피해야 할 아이템 |
| `[내 옷 확인하기]` | 카드의 `check_criteria`(기준 JSON) — `check_outfit` 도구로 넘겨 개인 착장 검사 (stateless 라 클릭형 버튼 아님) |

## 개인 착장 검사

카드의 `check_criteria`(기준 JSON)를 기준으로 **참석자 개인**이 자기 착장을 텍스트로 묘사
(또는 공개 `image_url`)해 `check_outfit` 으로 **비공개** 점검한다.

- **비공개 원칙** — 검사 결과는 본인에게만 보인다. 사진/착장 묘사/결과를 단톡방·주최자에게
  공유·저장하지 않는다 (이미지 입력 시 처리 후 즉시 폐기·미로깅 — [image-input.md](image-input.md) F7).
- 결과는 "가이드에 부합 / 일부 조정 권장 / 부적합" 정도의 피드백 + 이유.
- 조정이 필요하면 [착장 추천 플로우](matching-flow.md)(v1)로 대안을 제시할 수 있다.
- 입력 방식 상세는 [image-input.md](image-input.md), 도구 매핑은 [features.md](features.md) 참고.

## 확장: 톡캘린더 연동

- 톡캘린더 일정(시간/장소/모임명)을 가져와 모임장 입력 단계를 자동화.
- 일정 알림 시점에 맞춰 가이드 카드를 미리 띄워 주는 식으로 결합 가능.

> ⚠️ 위 카드 형식·문구·액션 라벨은 예시이며 확정 전이다.
> MCP 도구로 구현할 때의 입력/출력 스키마는 별도 설계 문서에서 정한다.
